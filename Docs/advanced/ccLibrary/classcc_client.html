Detailed Description

A BNCS Client class.

This is a class that provides full client functionality for BNCS.

Simple usage example
#include <ccClient.h>
class myClient : public IccClientCallback
{
public:
myClient()
{
m_client = new ccClient( this );
m_client->connect();
m_client->revertiveRegister( 101, 1, 10 );
m_client->infoWrite( 101, 1, "hi there");
};
~myClient() { delete m_client; };
protected:
// reimplement those functions we need to (declared in IccClientCallback)
virtual void cccInfoRevertive( int device, int index, const string & value )
{
Debug( "Infodriver Revertive %d %d '%s'", device, index, value.c_str() );
};
private:
ccClient *m_client;
}
The example shows our own client implementation "myClient" which inherits the callback class so that it can be notified of events generated by the ccClient class. We've only implemented a single event in this example - the Infodriver Revertive
Note
Watch out for parameter order (compared to other versions of BNCS Clients). The only thing you probably really need to remember about this is that index always follows device. e.g. routerCrosspoint( device, index, source ); infoWrite( device, index, slotContents ); This then matches the change of state callbacks e.g. cccInfoRevertive( device, index, slotContents );
