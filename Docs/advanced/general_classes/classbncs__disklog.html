Detailed Description

This is a simple Log-to-disk class.

There are many instances where you want to be able to simply log a message to the disk

This class puts logs in the logs directory under the %CC_ROOT%\%CC_SYSTEM% path (these are environment variables used for a standard V4.5 installation)

Logs can be written as a single ever-growing single file or as multiple timestamped files.

If you use a timestamped file it's anticipated that an external application will delete unwanted files. This class does nothing but create them!

Logs are written to a path given when you construct a bncs_disklog object - this is so that you can run multiple logs for multple purposes.

Log File Example:
The format of the log files is the same regardless of the timestamping options selected
14/08/2004 22:45:48: hello mum
14/08/2004 22:46:30: hello mum
14/08/2004 22:47:34: baarf
14/08/2004 22:47:34: hello mum
Note
The last file is left open until the bncs_disklog object is destroyed. For timestamped files this will be the last file written to (not necessarily the current file if there has been no recent writes). Open files are shared for reading but not writing.
Examples
// The following examples use CC_ROOT of "c:\bncs" and CC_SYSTEM of "v4.5"
// write "hello mum" to a daily timestamped file e.g. c:\bncs\v4.5\logs\mything\20040814.log
bncs_disklog log1( "mything", bncs_disklog::TIMESTAMP_DAY );
log1.write( "hello mum" );
// write "hello mum" to an hourly timestamped file e.g. c:\bncs\v4.5\logs\mything\20040814-09.log
bncs_disklog log2( "mything", bncs_disklog::TIMESTAMP_HOUR );
log2.write( "hello mum" );
// write "hello mum" to a plain file e.g. c:\bncs\v4.5\logs\mything\mything.log
bncs_disklog log3( "mything" ); // SINGLE is the default.....
log3.write( "hello mum" );

const char * bncs_disklog::ref ( void  ) 
Return reference (date/time) to the log just written

This is useful in telling the user a reference so that a developer may find the log entry easily on disk

Returns
reference string (not valid between bncs_disklog calls)
Example
bncs_disklog log( "mything" );
char *p = log.ref(); // p points to string of format "14/08/04 12:08:20 "

If you don't want to give a user an impossble to decipher error message tell them to log this reference for the developer to investigate later
