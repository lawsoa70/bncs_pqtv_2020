Detailed Description

This is a simple XML data access class.

Much data is stored in XML files and this class gives very quick and simple method of accessing that data. This is not a fully featured XML class - it is deliberately simple to use and therefore less fully featured.

This class also implies something about the structure of the XML file - the one key thing is that the XML file must contain elements that contain an "id" attribute that uniquely identifies them.

Example data used throughout this document
Consider the following simplified XML document "instances.xml" stored in the <CC_ROOT>\<CC_SYSTEM>\config folder
<instances>
<instance id="ARC1" type="ARC2000">
<instanceInfo device="223" offset="100" />
</instance>
<instance id="ARC Chain" type="ARCOverview" composite="yes" >
<group id="main" instance="ARC1" />
<group id="reserve" instance="ARC2" />
<group id="another" instance="ARC3" />
</instance>
</instances>
Simple Examples
Lets say I want to find out the "type" of ARC1. This is the code required:
bncs_config c( "instances.ARC1" );
bncs_string a = c.attr( "type" ); // for the above example a contains "ARC2000"
Lets say I want to find out the "device" number within the "instance" for a device instance of ARC1. We know from the data that there will only ever be one child "instance" tag. This is the code required:
bncs_config c( "instances.ARC1" );
bncs_string d = c.childAttr( "device" ); // for the above example s contains 223
The dotted notation of the bncs_config constructor tells us the file to look for is instances.xml, and then to search for an entry whose id is "ARC1".
Note
The important thing to note is is that every level we want to be able to uniquely identify MUST have an "id" attribute
Going down more levels
Lets say I want to find out the "instance" for the group "reserve" for "ARC Chain". You'd use code that looks like this:
bncs_config c( "instances.ARC Chain.reserve" );
bncs_string s = c.attr( "instance" ); // for the above example s contains ARC2
The first part of the constructor string says look in "instances.xml". The second says "find me a child uniquely identified by "ARC Chain". The third says "find me a child of "ARC Chain" uniquely identified by "reserve".
Cacheing
For efficiency this function caches the last 100 files that its been asked to load. This number of files is subject to change. This also means that a change made to a file whilst an application is running may not be available since the application will still be using the cached version.

bncs_config::bncs_config ( const bncs_string &  name) 
Constructs a bncs_config using the file/keyname supplied
Parameters
namepath and key to locate (see below)

The name is made up of one or more components, separated by periods.

Example:
bncs_config s( "instances.mything.subthing" );
The First part of the key name is the file in which to look. The environment variables CC_ROOT and CC_SYSTEM are used to determine the root path for this file and configuration files are always to be found in the config subdirectory. So, this example looks in a file <CC_ROOT>\<CC_SYSTEM>\config\instances.xml

The subsequent parameters are optional and determine the number of levels down the XML tree to search the "id" attributes.

void bncs_config::drillDown ( void  ) 
Move down to the next child element
Example: (using example data above)
bncs_config c( "instances" );

This leaves the current and child elements like this:

current---> <instances>
child ---> <instance id="ARC1" type="ARC2000">
<instanceInfo device="223" offset="100" />

calling drillDown() from this poisition leaves them here:

<instances>
current---> <instance id="ARC1" type="ARC2000">
child ---> <instanceInfo device="223" offset="100" />
Example
bncs_config c("instances" );
bncs_string s1 = c.childAttr( "id" ); // s1 contains "ARC1"
c.drillDown(); // move down one level in the XML
bncs_string s2 = c.childAttr( "device" ); // s2 contains "223"
c.drillDown();
bool b1 = c.isValid(); // b1 contains true (pointing at <instanceInfo device="223" offset="100" />)
bool b2 = c.isChildValid(); // b2 contains false - current entry does not have any children
Note
You can use isValid() and isChildValid to determine the success of this command
