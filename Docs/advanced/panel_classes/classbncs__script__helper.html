Detailed Description

The base class for QtUI C++ scripts.

This class is a means of writing simple C++ programs to "script" BNCS.
This class can be used to create top level "panels" or write simple components to be used as "smart buttons" on panels

You aren't expected to write applications by hand using this class - there is a Wizard application that will create the applications for you.

Note
Important information for ApplCore users:
All button push notifications come back to buttonCallback()
All network revertive notifications come back to revertiveCallback()
All database change notifications come back to databaseCallback()
ET (executive timer) calls are now timerStart() and timerStop(). Notifications come back to timerCallback()
EC is now captionEnable()
EB is partly replaced by setGeometry() and setSize()

void bncs_script_helper::actionAll ( const bncs_string &  sValue) 
Set all of the alarm controls to a specified state.

The writing of this method was motivated by the CCM project where it is a common use case to force all alarm controls on a panel to the same state or to acknowledge all alarms on a panel by pushing a single button. This method collects all controls with identifiers starting with "alm_" and sets them to the specified state. When the specified state is "acknowledge", then only controls that are not in off, acknowledged, forced or ignored state will be set to acknowledged. The method supports single popup-buttons as well as script components. When a script component is to be addressed by this method, then it will be necessary to support the "alarm" command by the component.

Parameters
sValueThe value to be set. It must be a number as specified in <bncs_alm_common.h>, or "ack"

void bncs_script_helper::enableActionAll ( ) 
Enables broadcasting alarm state to all controls with identifiers starting with "_alm".

This method places a trigger in the "callback" method of this class. As soon as a notification is received form a button or component with identifier "ActionAll" or "AckAll", the trigger will call actionAll() that will broadcast a message to all controls with identifier starting with "alm_" on the panel.

See actionAll()
