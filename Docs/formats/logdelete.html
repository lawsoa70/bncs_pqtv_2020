Overview

The logdelete.xml file details what folders the delete functionality will act upon. If there is not a workstation specific setting the global settings will be used instead.

Attribute

Usage

Comments

id

Either the workstation or glabal

 

enable

If not true the logdelete function will not be used.

 

daysback

Files older than this number of days will be deleted

 

default

If false the default files will not be deleted, these are the current logs directory and the caplogs directory.

 

name

The name of additional directories to have old files that require deleting, if they are not fully qualified they will be assumed to be in the current system.

 

Sample Contents - /config/logdelete.xml

 

DTD

<?xml version="1.0" encoding="UTF-8"?>
<!ELEMENT logdelete (workstation*)>
<!ELEMENT workstation (dir*)>
<!ATTLIST workstation
        id CDATA #REQUIRED
        enable CDATA #REQUIRED
        daysback CDATA #REQUIRED
        default CDATA #IMPLIED
> 
<!ELEMENT dir EMPTY>
<!ATTLIST dir
        name CDATA #REQUIRED
>
