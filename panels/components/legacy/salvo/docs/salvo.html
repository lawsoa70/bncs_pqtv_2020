Salvo

Description

This component is used to send a sequence of crosspoint commands defined in the config file /config/salvoes.xml

The format of the XML file is as follows:

<salvoes>
    <salvo id="jf_misc_1-1_2-2" instance="rtr_audio_analogue" >
        <item source="172" dest="66" />
        <item source="173" dest="67" />
    </salvo>
    <salvo id="jf_misc_3-1_4-2" instance="rtr_audio_analogue">
        <item source="174" dest="66" />
        <item source="175" dest="67" />
    </salvo>
    <salvo id="jf_misc_1-3_2-4" instance="rtr_audio_analogue" >
        <item source="172" dest="68" />
        <item source="173" dest="69" />
    </salvo>
    <salvo id="jf_misc_3-3_4-4" instance="rtr_audio_analogue">
        <item source="174" dest="68" />
        <item source="175" dest="69" />
    </salvo>
</salvoes>

  • Each salvo element must have a unique id attribute and this is used by the component as a reference
  • The instance attribute defines which router device instance the crosspoint commands should be sent to
  • The item child elements have two attributes that contain the source and destination for each crosspoint
The component works with a packager, router or a router driven via an infodriver. This operation is determined by the "type" of the instance as per the following:

Commands

Name Type Use
label mandatory The text to be shown on the salvo button
salvo mandatory The ID of the salvo as defined in salvoes.xml
fire run-time method This command may be issued via connections to cause the salvo to fire
e.g. when a slave salvo should fire after a master salvo has fired - see Developers Notes below

Notifications

Name Use
fired=<salvo> notification that the salvo has been fired

Stylesheets

Name Use
salvo_active appearance when active
- indicates that all destinations have the source defined in the salvo currently routed
salvo_inactive appearance when inactive
- indicates that one or more destinations do not have the source defined in the salvo

Developers Notes

To fire a salvo when another salvo fires the recommended connection is shown below:

Sender: another_salvo    Receiver: this_salvo    Event: fired=*    Command: fire=[value]

In addition to the salvo_active/salvo_inactive states are applied to the button it may also be useful to show the current source on each of the destinations that are routed by the salvo.
The tally component can be used to show the source that is currently routed to any destination and these can be added to the panel then arranged and labelled as required.
