Tab Component

The Tab component can be used to add tabs to a panel without the need for scripting

If your panel has more information than can fit on one page than this component maybe for you.

How to use

Place the tab component on the Main UI page, ensure the UI has the same name as the folder. Stretch the component out to the fill the whole panel, this will be the only thing on the page. Now fill in the parameters for the component, the descriptions for these can be found in the Parameters Section below.

Add extra UIs to the folder make sure they are named the same as the names given to them in tab_n_Name parameters, one UI for each page that is needed. Open "Edit Dialog Properties" for each of these extra pages and set the X Pos and Y Pos as necessary to stop panel from hiding the tab when it is shown (In most case setting X Pos to 50 should do it, if the tabs are to appear at the left hand edge).

Parameters

Parameter Example Use
layout 2xtabs Select which tab layout to use, these can be found in components/tab/tabs. Once selected click Ok to close the paramenters and then open it again, as this will refresh the Parameters with new parameter list for the tab layout you selected.
panel_location studio_1\umd_naming This is the location of the folder that UIs are in. The location starts from Panels\ so only the directory structure after this has to be added.
tab_N Main_tab The Name of the UI to be displayed, must be in the folder specified in panel_location
tab_N_Name Main The Name displayed button for this UI
The Above 2 rows will be shown as many time as there are tab buttons, with N replace with a tab number
firt_tab_shown tab_1 The tab number to be shown first of page load

Creating a New Tab Layout

Creating a New Tab Layout with more buttons or a different style is very easy. The easiest way to do this is to copy an already exisiting UI, all the tab UI can be found in panels/components/tab/tabs.

Each tab button needs the ID tab_N(where N is the number of the tab), this is also the case for the backgound item for the tab, which is called bg_tab_N. As long as this naming convention is followed the component will be able to display the correct number of parameters needed for the panel.

Example: A 3 tab layout, would have IDs named tab_1,tab_2,tab_3 and backgrounds IDs bg_tab_1,bg_tab_2,bg_tab_3

FAQ

Problem Solution
I cannot see the tabs, only the first page The Likely casue of this is X Pos OR Y Pos not being set for the UI being shown, UI is covering the Tabs. Open up the UI that is hidding the tab and right click on background of the UI, you will be presented with a "Edit Dialog Properties" Windows, change the X Pos Or Y Pos in here to stop the tabs beening hidden (In most case setting X Pos to 50 should do it, if the tabs are to appear at the left hand edge).
Not enough Parameters are shown for the layout I have selected This is probably because the paramlist has not had a chance to refresh, click OK on the param dialog to close it and than open it again. It should now show the correct number of parameters. If this is not the case click the Layout select follows the instructions for creating a New Tab Layout.
